# 图片处理工具

一个纯前端、可批量处理的图片工具，聚焦四个高频场景：

- 图片压缩
- 图片去背景
- 图片加水印
- 图片改尺寸

项目界面风格与当前线上演示保持一致，可直接参考：

- 在线预览：[https://imgedit.ykxai.top/](https://imgedit.ykxai.top/)

## 项目简介

这是一个不依赖后端图片上传的静态网页应用。用户在浏览器中选择图片后，压缩、去背景、水印处理都在本地完成，适合：

- 给运营、设计、内容同学快速处理图片
- 对外提供一个轻量、零安装的图片工具页面
- 内网或外网部署后直接给其他用户访问

项目当前实现以“简单、直接、可批量处理”为主，不走复杂工作流，不需要登录，也不需要构建步骤。

## 核心特性

### 1. 图片压缩

- 支持 `PNG / JPG / WebP` 导入
- 支持导出为 `WebP / JPG`
- 可调整清晰度
- 可限制最大边长
- 上传 1 张时处理单张，上传多张时自动批量处理
- 下载结果会自动判断：
  - 单张：直接下载图片
  - 多张：下载 ZIP

### 2. 图片去背景

- 支持导出为 `PNG / WebP / JPG`
- 支持背景容差调节
- 支持最大边长控制
- 支持批量处理
- 支持对当前图片撤销

说明：当前去背景采用浏览器端的轻量算法，适合纯色、浅色、边界较清晰的背景图，不是 AI 级精细抠图。

### 3. 图片加水印

- 支持文字水印
- 支持导出为 `WebP / JPG / PNG`
- 支持颜色、透明度、大小调整
- 支持两种铺法：
  - 单个位置
  - 全屏随机
- 全屏随机模式支持自定义水印数量
- 支持批量处理
- 支持对当前图片撤销

### 4. 图片改尺寸

- 支持等比缩放
- 支持按 `宽度 / 高度` 调整
- 自动保持原始宽高比
- 上传 1 张时处理单张，上传多张时自动批量处理
- 默认保持当前图片格式输出

## 隐私与处理方式

这是本项目最重要的一点：

- 图片不会主动上传到服务器
- 图片处理逻辑运行在用户自己的浏览器里
- 即使网页部署在外网，只要你没有额外接入上传接口，原图和处理结果仍然只在用户本地浏览器中参与处理

适合对隐私比较敏感、又希望把工具开放给其他人直接访问的场景。

## 界面与交互

项目采用单页四标签结构：

- `图片压缩`
- `去背景`
- `加水印`
- `改尺寸`

交互设计重点：

- 顶部菜单切换不同业务功能
- 左侧参数区只展示当前业务的操作项
- 右侧展示当前图片与处理队列
- 支持拖拽导入与批量导入
- 队列中可切换当前图片、查看大小变化、单独下载结果

## 技术实现

项目不依赖框架，直接使用原生前端技术实现：

- `HTML`：页面结构
- `CSS`：界面样式
- `JavaScript`：图片处理、状态管理、下载逻辑
- `Canvas`：压缩、去背景、水印绘制
- `Blob / ObjectURL`：本地预览与下载
- 原生 ZIP 打包逻辑：批量下载结果

### 当前处理策略

- 压缩：通过 `Canvas` 重绘并导出为目标格式
- 去背景：基于四角取样 + 色差容差进行透明化处理
- 水印：在 Canvas 上绘制文字，支持固定位置与随机散布
- 改尺寸：按目标宽度或高度进行等比缩放

## 本地运行

项目是纯静态页面，不需要安装依赖。

### 方式一：直接用静态服务启动

```bash
python -m http.server 3013
```

然后访问：

```text
http://127.0.0.1:3013/
```

### 方式二：使用 PM2 启动

如果你希望长期运行，推荐直接用 PM2 托管静态目录：

```bash
pm2 serve . 3013 --name image-tool
```

常用命令：

```bash
pm2 status
pm2 logs image-tool
pm2 restart image-tool
pm2 stop image-tool
pm2 delete image-tool
```

## 使用说明

### 图片压缩

1. 打开页面
2. 拖入或选择图片
3. 选择导出格式
4. 调整清晰度和最大边长
5. 点击 `开始压缩`
6. 点击 `下载结果`

### 去背景

1. 切换到 `去背景`
2. 导入图片
3. 调整背景容差
4. 如有需要设置导出格式与最大边长
5. 点击 `去背景当前` 或 `批量去背景`
6. 对当前结果不满意可点击 `撤销当前`

### 加水印

1. 切换到 `加水印`
2. 输入水印文字
3. 选择颜色、透明度、大小
4. 选择铺法：
   - 单个位置
   - 全屏随机
5. 如果选择全屏随机，可设置随机数量
6. 点击 `加水印当前` 或 `批量加水印`

### 改尺寸

1. 切换到 `改尺寸`
2. 导入图片
3. 选择按宽度或按高度调整
4. 输入目标像素
5. 点击 `开始调整`
6. 点击 `下载结果`

## 常见问题

### 1. 为什么有些图片压缩后反而变大了？

常见原因有几个：

- 原图本身已经压得比较狠
- 原图尺寸不大，重新编码反而增加了开销
- `PNG` 转成高质量 `JPG / WebP` 时参数过高
- 图片颜色简单、透明区域多，某些格式未必更占优

建议：

- 优先导出 `WebP`
- 适当降低清晰度
- 结合“最大边长”一起使用

### 2. 外网访问时，图片会传到服务器吗？

不会。当前项目没有服务端上传接口，处理逻辑在浏览器本地执行。

### 3. 去背景为什么对复杂背景效果一般？

因为当前实现是轻量浏览器算法，不是深度学习分割模型。它更适合：

- 纯色背景
- 商品白底图
- 主体边界较清晰的图片

### 4. 为什么批量下载是 ZIP？

因为浏览器对多文件连续下载限制较多，批量场景下统一打包成 ZIP 更稳定。

## 当前限制

- 去背景不是 AI 精抠图
- 处理超大图时会受到浏览器内存限制
- 当前不支持 PSD、GIF、AVIF、HEIC 等格式
- 水印目前仅支持文字，不支持图片水印
- 改尺寸当前仅支持等比缩放，不支持裁切与自由拉伸
